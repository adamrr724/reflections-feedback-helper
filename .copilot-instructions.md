## MANDATORY REQUIREMENT: Always Use Official Questions

### When helping with REFLECTIONS:
**CRITICAL:** ALL reflection work MUST be based on the official FY26 reflection questions from `/reflection/reflection-questions.txt`

**The ONLY acceptable reflection structure is:**
1. **What results did you deliver, and how did you do it?**
   - Quantify your impact with measurable results
   - Leverage of AI applications  
   - Security contributions
   - Culture in action (GitHub values: Customer-Obsessed, Ship to Learn, Growth Mindset, Own the Outcome, Better Together, Diverse & Inclusive)
   - People Management (if applicable)

2. **Reflect on recent challenges - what did you learn and how did you apply a growth mindset?**
   - Describe the situation and why it tested skills/mindset
   - Describe actions taken and openness to learning/feedback
   - Describe how the experience shaped thinking/approach

3. **What are your goals for the upcoming period?**
   - Define 3-5 SMART goals with business outcomes focus
   - Goals should support secure, scalable products/services
   - Include people management goals if applicable

### When helping with FEEDBACK:
**CRITICAL:** ALL feedback MUST use the 3-question template from `/feedback/questions.txt`

### Assistant Guidelines for REFLECTIONS:
1. **FIRST STEP:** Always read and reference `/reflection/reflection-questions.txt` for exact question wording
2. **STRUCTURE:** Answer each official question directly - do not create custom templates
3. **EVIDENCE:** Use specific data from `/reflection/contributions/` folder
4. **VALUES:** Incorporate all six GitHub values as specified in the official questions
5. **FORMAT:** Follow the official question format exactly, not project impact templates
6. **NEVER:** Create alternative reflection structures or skip official questions

### Assistant Guidelines for FEEDBACK:
1. **MANDATORY:** Use only the 3-question format from `/feedback/questions.txt`
2. **Check recipient files** in `/feedback/recipients/` for context and history
3. **Structure:** Answer each of the 3 official feedback questions directly
4. **Tone:** Professional, constructive, and actionable
5. **Specificity:** Include concrete examples and observations
6. **NEVER:** Create alternative feedback formats

### Content Creation Guidelines:
- **Be evidence-based** - Use specific examples from contributions data
- **Answer questions directly** - Don't reorganize or reframe official questions  
- **Reference existing materials** for context and consistency
- **Quantify impact** wherever possible using data from contributions folder
- **Maintain professional tone** appropriate for workplace feedback
- **ALWAYS verify** you're using the official questions before starting any work

### Writing Style for Final Outputs:
- **Write naturally** - Sound like a real person, not a corporate robot
- **Use paragraphs** - Avoid excessive bullet points in final reflections and feedback
- **Be conversational** - Professional but authentic, not overly formal
- **Cut fluff** - Remove superfluous language, buzzwords, and corporate speak
- **Show personality** - Let the writer's voice come through while staying professional
- **Connect ideas** - Use natural transitions between thoughts, not just lists
- **Be concise** - Make every sentence count, avoid redundant phrasing

### File Management:
- **Preserve original notes** - Never overwrite user's working notes
- **Use consistent naming** - Follow existing folder structure
- **Separate drafts from finals** - Keep working notes distinct from polished versions
- **Maintain organization** - Contributions data separate from reflection drafts
