## サポートエンジニア、日本語サポート (リモート - アジアパシフィック)

GitHubでは、アジアパシフィック地域を担当するサポートチームにおいて、日本のお客様に日本語での技術サポートを提供する、フルタイムのサポートエンジニアを募集します。

企業のお客様によるGitHub Enterpriseの導入事例が増えるにつれ、GitHubのサポートチームも大きくなっています。エンタープライズサポートチームは、企業のお客様から運用上の技術的な問い合わせを受け付けます。私たちのゴールは、こうしたお問い合わせに可能な限り迅速かつ丁寧に回答し、お客様を問題解決へ導くことです。

私たちは、アジアパシフィック地域を担当するチームの一員として日本語の技術サポートを提供できるエンジニアを募集しています。このポジションはリモート勤務ですが、応募する場合には日本、オーストラリア、ニュージーランド、インドのいずれかの国に居住しており、その国での就労資格を持っている必要があります。

このポジションの主な職務内容は、日本のお客様によってウェブインターフェースを通じて起票されるサポートチケットを通して、日本語で問題解決を行うことです。サポートチケットは通常、企業のシステム管理者の方によって起票されます。その内容はGitHub Enterprise Serverインスタンスの負荷に関する調査依頼や、ネットワークの問題に関する調査、アプリケーションエラーのデバッグ、機能リクエストへの対応など、多岐に渡ります。

また同時に、エンジニアリングチームと連携して製品開発や課題の優先順位付けを行ったり、ドキュメントチームと連携してお客様向けに公開されたヘルプドキュメントの改善なども行います。決してGitのエキスパートである必要はありませんが、技術面での経験と、人を助けることに対する情熱を持っている必要があります。

このポジションはサポートマネージャーと共に、社内の他チームやリーダーたちと連携して業務を遂行します。

GitHubでは、社員が可能な限りリモートかつ非同期に仕事を進める文化が定着しています。したがって東京オフィスの近くに居住している必要はありませんが、社内イベントやコワーキングのために年に数回程度の出張があります。

このポジションの主な職務内容は、日本のお客様に対して、日本の営業時間帯に技術サポートを提供することですが、他のアジアパシフィックチームのメンバーと同様、日本以外のアジアパシフィック地域のお客様に対して英語でサポートを提供する機会もあります。

なお、応募の際のフォームの記入や履歴書の提出は英語のみ受け付けています。採用のプロセスには、日本語と英語両方での面接が含まれます。

- Linuxサーバ、ソースコード、ウェブアプリケーションに関する複雑なサポートチケットの解決
- お客様と社内のエンジニアリングチーム両方に対する、技術的内容の円滑な意思疎通
- お客様に対する、製品のアップグレードや新機能に関する情報、安定した運用のためのアドバイスなどの積極的な提供
- GitHub Enterpriseのコードベースにおけるバグや改善点の発見
- お客様の営業時間中に発生した緊急性の高い問題に対する対応
